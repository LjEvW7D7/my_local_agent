# 🤖 Локальный ИИ-Агент «Кот» с интеграцией Google Календаря и Браузера

Умный локальный ИИ-ассистент на базе модели **Qwen 2.5 Coder 14B**, запущенный через **Ollama**. Агент обладает долгосрочной памятью, умеет управлять локальной файловой системой, искать информацию в интернете в обход блокировок роботов и контролировать расписание пользователя в Google Календаре с функцией всплывающих Windows-напоминаний.

---

## 🔥 Основные возможности
* 🗂 **Управление файлами**: умеет сканировать, читать, создавать и дополнять файлы в изолированной папке `workspace/`.
* 📅 **Google Календарь API**: синхронизирует расписание, выводит список дел на русском языке, создаёт и удаляет события.
* 🔔 **Windows Напоминания**: фоновый поток каждые 60 секунд проверяет календарь и выводит системное окно `MessageBox` за 30 минут до начала задачи.
* 🕸 **Скрытый веб-скрейпинг**: использует `Playwright` и `playwright_stealth` для безопасного чтения сайтов (включая Google Поиск) в обход капчи и защитных экранов.
* 💾 **Долгосрочная память**: сохраняет контекст переписки в JSON-файл между перезапусками терминала со встроенной защитой от переполнения контекста.

## 🛠 Структура проекта

Для работы агента в корневой папке (например, `D:\local_AI_Agent\`) должны находиться следующие файлы:

```text
local_AI_Agent/
│
├── agent.py               # Основной исходный код агента на Python
├── credentials.json       # Секретный ключ авторизации Google API (создаете вы)
├── token.json             # Токен сессии Google (создается автоматически при первом запуске)
├── chat_history.json      # База данных памяти диалога (создается автоматически)
└── workspace/             # Изолированная рабочая папка, куда ИИ пишет код и файлы
    ├── books.txt          # Пример созданного агентом файла
    └── test.py            # Пример написанного агентом скрипта
```

## 📦 Системные требования и установка

### 1. Установка и запуск модели
1. Скачайте и установите [Ollama](https://ollama.com).
2. Запустите терминал и скачайте модель команды:
   ```bash
   ollama run qwen2.5-coder:14b
   ```
3. Убедитесь, что Ollama запущена в фоновом режиме (по умолчанию доступна на `http://localhost:11434`).

### 2. Установка зависимостей Python
Проект протестирован на **Python 3.13**. Установите необходимые библиотеки:
```bash
pip install requests playwright playwright-stealth google-auth-oauthlib google-api-python-client
```

После установки `playwright` обязательно инициализируйте системные бинарники браузеров:
```bash
playwright install chromium
```

## 🔑 Инструкция: Как получить API токен для Google Календаря

Чтобы агент мог подключаться к вашему календарю, необходимо создать проект в консоли разработчика Google:

1. **Создание проекта**:
   * Перейдите на [Google Cloud Console](https://google.com).
   * Нажмите **Select a project** -> **New Project**, укажите имя (например, `Local-AI-Agent`) и нажмите **Create**.

2. **Включение Calendar API**:
   * В левом меню выберите **APIs & Services** -> **Library**.
   * В строке поиска введите **Google Calendar API**.
   * Кликните на него и нажмите **Enable**.

3. **Настройка экрана согласия (OAuth Consent Screen)**:
   * Перейдите в **APIs & Services** -> **OAuth consent screen**.
   * Выберите тип **External** и нажмите **Create**.
   * Заполните обязательные поля: *App name* (Кот Агент), *User support email* (ваш Gmail) и *Developer contact information* (ваш Gmail). Нажмите *Save and Continue*.
   * В блоке **Scopes** нажмите *Add or Remove Scopes*, найдите и выберите `.../auth/calendar` (доступ к управлению календарем).
   * В блоке **Test users** обязательно нажмите **Add Users** и введите ваш Gmail-адрес. *Без этого Google заблокирует вход!*

4. **Создание Ключей (Credentials)**:
   * Перейдите во вкладку **APIs & Services** -> **Credentials**.
   * Нажмите кнопку **+ Create Credentials** -> **OAuth client ID**.
   * В выпадающем списке *Application type* выберите **Desktop app**.
   * Нажмите **Create**.
   * В появившемся окне нажмите **Download JSON**.

5. **Размещение в проекте**:
   * Переименуйте скачанный файл в **`credentials.json`**.
   * Перенесите его в корневую папку рядом со скриптом `agent.py`.

## 🚀 Запуск и использование

Запустите агента из терминала:
```bash
python agent.py
```

### Первый запуск
При первом обращении к функциям календаря скрипт автоматически откроет вкладку в вашем браузере. Выберите ваш тестовый Gmail-аккаунт, нажмите *«Дополнительно» -> «Перейти к небезопасному приложению»* (это стандартное предупреждение для ваших личных скриптов) и подтвердите разрешения. После этого в папке создастся файл **`token.json`**, и авторизация больше не потребуется.

### Примеры команд для общения с Котом:
* *«Что у меня запланировано на сегодня?»* (Выведет список встреч на русском языке).
* *«Создай встречу Практика по Golang сегодня в 20:00 на полтора часа»*.
* *«Посмотри, какие файлы есть в workspace»*.
* *«Прочитай файл test.py и перепиши его, добавив обработку ошибок»*.
* *«Изучи страницу https://github.com и расскажи, что там интересного»*.

Для завершения работы введите в консоль `exit`, `quit` или `выход`.

# 🤖 Локальный ИИ-Агент «Кот» с интеграцией Google Календаря и Браузера

Локальный ИИ-агент на базе Ollama (модель `qwen2.5-coder:14b`), который выступает
персональным наставником по Python/Golang/AI/Cyber Security и одновременно
управляет календарём, задачами и файлами пользователя. Есть консольный режим
и веб-интерфейс на Flask.

## 1. Структура файлов

| Файл | Роль |
|---|---|
| `mentor_core.py` | Ядро: вся логика агента, инструменты, работа с Google Calendar, фоновые напоминания |
| `app.py` | Flask-обёртка вокруг `mentor_core` — HTTP API + отдача `index.html` |
| `index.html` | Одностраничный веб-клиент (чат + боковая панель: события/задачи/прогресс) |
| `chat_history.json` | Персистентная история диалога (перезаписывается при каждом сообщении) |
| `tasks.json` | Хранилище задач/напоминаний |
| `credentials.json` / `token.json` | OAuth-креды и токен доступа к Google Calendar (см. предупреждение выше) |
| `books.txt` | Статичный список книг по Python/Golang для рекомендаций |
| `main2.py`, `test.py` | Разрозненные тестовые скрипты, не связаны с ядром |
| `Текстовый_документ.txt` | Тестовый файл ("привет мир") |

## 2. Архитектура

```
Пользователь
   │
   ├── консоль (start_chat_agent)  ИЛИ  веб (app.py, Flask)
   │
   ▼
process_message(chat_history, user_input)
   │
   ├── ask_ollama_chat()  →  локальный Ollama (localhost:11434)
   │
   ├── extract_json_blocks()  →  парсит JSON-команды из ответа модели
   │
   ├── выполнение инструментов (если найдены валидные action):
   │     calendar / tasks / progress / files / browser
   │
   └── повторный запрос к модели → финальный ответ пользователю
```

Ключевая деталь: модель не вызывает функции через tool-calling API, а сама
генерирует JSON-блоки в тексте ответа, которые затем находит
`extract_json_blocks` (парсер со счётчиком вложенности скобок, устойчивый
к `{`/`}` внутри строк — например, в содержимом `write_file`).

## 3. Инструменты агента

| Действие | Назначение |
|---|---|
| `create_event` / `delete_event` / `list_events` | Google Calendar |
| `visit_url` | Скрытый браузер (Playwright + stealth) — читает текст страницы |
| `list_files` / `read_file` / `write_file` | Работа с файлами в песочнице `./workspace` |
| `create_task` / `list_tasks` / `complete_task` / `delete_task` | Лёгкие задачи/напоминания (аналог Mira) |
| `update_progress` / `get_progress` | Учебный прогресс по 4 темам (уровень 0–5, статус, заметки) |

Дополнительно есть два фоновых потока-напоминалки:
- `calendar_reminder_worker` — уведомление за 30 минут до события в календаре;
- `task_reminder_worker` — уведомление за 30 минут до дедлайна задачи и при просрочке.

Оба используют `ctypes` + `MessageBoxW`, то есть **работают только на Windows**.

## 4. Веб-интерфейс (`app.py` + `index.html`)

- `GET /` — отдаёт `index.html`.
- `POST /api/chat` — принимает сообщение, вызывает `core.process_message`, отдаёт ответ.
- `GET /api/state` — данные для боковой панели (задачи, ближайшие события, прогресс).
- `GET /api/history` — история чата при загрузке страницы (без системного промпта и служебных сообщений).

Интерфейс — тёмная тема с шрифтами Source Serif 4 / Inter / JetBrains Mono,
русскоязычный, с индикатором "думает…" и авто-обновлением боковой панели раз в минуту.

## 5. Наблюдения и потенциальные проблемы

1. **Безопасность credentials.json / token.json.**
   Реальные секреты лежат открытым текстом рядом с кодом. Рекомендуется вынести
   их за пределы репозитория/переписки, добавить в `.gitignore`, и — раз они уже
   были кому-то показаны — сбросить client secret и токен в Google Cloud Console.

2. **Кросс-платформенность.** `send_windows_notification` использует
   `ctypes.windll`, которого не существует вне Windows — на Linux/macOS фоновые
   напоминания будут падать с исключением на каждой итерации (перехватывается,
   но напоминания просто не будут работать).

3. **Мусорные данные от предыдущих ошибок.** В `chat_history.json` и `tasks.json`
   видно последствие бага: модель однажды подставила в JSON-команды литерал
   `"ГГГГ-ММ-ДДTЧЧ:ММ:СС"` вместо реальной даты (не разобралась, что это
   плейсхолдер в системном промпте, а не пример готового значения). В итоге:
   - создалась мусорная задача `«Текст задачи»` с "датой" — строкой-подсказкой;
   - не создалось событие в календаре (`Invalid isoformat string`);
   - не удалось открыть сайт `https://полный адрес сайта`.
   Стоит почистить `tasks.json` от этой записи и, возможно, переформулировать
   плейсхолдеры в системном промпте как явный текст-инструкцию, а не как
   значение внутри примера JSON (например: `"start": "<ГГГГ-ММ-ДД>T<ЧЧ:ММ:СС>, обязательно подставь реальную дату"`).

4. **`.pyc`-файл в проекте.** `mentor_core_cpython-314.pyc` — скомпилированный
   артефакт, обычно не нужен в репозитории/переписке рядом с исходником; вероятно,
   попал туда случайно из `__pycache__`.

5. **`progress.json` не был предоставлен**, хотя `mentor_core.py` явно на него
   ссылается (`PROGRESS_FILE`). Он создастся автоматически при первом вызове
   `update_progress`/`get_progress`, если его нет.

6. **Зависимости.** Судя по импортам, для запуска нужны: `flask`, `requests`,
   `playwright` (+ браузеры через `playwright install`), опционально
   `playwright-stealth`, `google-auth`, `google-auth-oauthlib`,
   `google-api-python-client`, а также локально запущенный `ollama serve`
   с моделью `qwen2.5-coder:14b`.

## 6. Идея / назначение проекта

Персональный русскоязычный обучающий агент-компаньон: ведёт пользователя по
четырём техническим темам, сам замечает в разговоре скрытые намерения
